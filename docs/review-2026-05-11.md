# Project Review

**Date:** 2026-05-11
**Scope:** full
**Project:** johnboen.com (portal)
**Reviewer:** Claude Code

## Executive Summary

The portal is a clean, well-structured static site that does its job effectively. The most significant findings are documentation drift (CLAUDE.md, README, and the design doc all disagree on project count and card details), dead CSS left behind after hero CTAs were removed, and a missing `.gitignore`. No security vulnerabilities were found — the site has no dynamic input handling, no secrets, and no dependencies beyond a Google Fonts stylesheet.

## Findings

### [MEDIUM] F-01: No `.gitignore` file

**Category:** Operations
**Evidence:**
- `ls -la` at project root shows no `.gitignore`
- Confirmed: `cat .gitignore` returns "No such file or directory"

**Impact:** Any IDE config files, OS artifacts (`.DS_Store`, `Thumbs.db`), or temp files created during local development will show up in `git status` and could be accidentally committed.
**Fix:** Add a `.gitignore` with standard entries for OS files, IDE configs, and common temp files. Minimal example:

```
.DS_Store
Thumbs.db
*.swp
*.swo
.vscode/
.idea/
```

### [MEDIUM] F-02: CLAUDE.md lists five project cards; site has six

**Category:** Documentation
**Evidence:**
- `CLAUDE.md:21` lists: "Certification, JobClass, AI Usage Trust Paradox, Story Structure Explorer, AI Usage Repository"
- `index.html` has six cards: Certification, JobClass, MD Reader, Epstein DOJ Disclosures, AI Usage Trust Paradox, Story Structure Explorer
- CLAUDE.md is missing MD Reader and Epstein, and lists "AI Usage Repository" which doesn't exist as a card

**Impact:** Anyone using CLAUDE.md as guidance (including Claude Code) will have a wrong mental model of the site's content.
**Fix:** Update CLAUDE.md section "Site Structure" item 2 to list the actual six cards by name.

### [MEDIUM] F-03: README project table is stale

**Category:** Documentation
**Evidence:**
- `README.md:11` says "five actively maintained projects"
- `README.md:19` Certification table row says "1,300+ scenario-based questions across 26 exams and 4 providers"
- Actual card (`index.html:60`) says "2,500 sample questions covering 50 exams across ten providers"
- README table is missing MD Reader and Epstein DOJ Disclosures entries

**Impact:** README is the first thing visitors see on GitHub. Stale numbers and missing projects misrepresent the current state.
**Fix:** Update README to list all six projects with current descriptions matching the live site.

### [MEDIUM] F-04: Design doc certification numbers don't match card

**Category:** Documentation
**Evidence:**
- `docs/project_design.md:168` says "4,600+ sample questions covering 46 exams across ten providers"
- `index.html:60` says "2,500 sample questions covering 50 exams across ten providers"
- These are different numbers in both directions (more questions in design doc, more exams on site)

**Impact:** No single source of truth for project stats. Future card updates may use the wrong baseline.
**Fix:** Verify current actual numbers from the certification repo, then update both the design doc and the card to match.

### [MEDIUM] F-05: Nav scroll-tracking JS adds `.active` class but CSS never styles it

**Category:** Consistency
**Evidence:**
- `js/main.js:39` adds `link.classList.add('active')` to `.nav-link` elements on scroll
- `css/styles.css` has no `.nav-link.active` rule — grep for `\.nav-link\.active` returns no matches
- The only `.active` styles are for `.nav-toggle.active` (hamburger animation)

**Impact:** The scroll-tracking feature runs on every scroll event but produces no visible result. Users get no indication of which section they're viewing.
**Fix:** Add a `.nav-link.active` CSS rule, e.g.:
```css
.nav-link.active {
    color: var(--color-text-primary);
}
```

### [LOW] F-06: Dead CSS — `.hero-ctas` and `.btn-primary` rules unused

**Category:** Dead Code
**Evidence:**
- `css/styles.css:246-250` defines `.hero-ctas`, `css/styles.css:267-277` defines `.btn-primary` and `.btn-primary:hover`, `css/styles.css:725-731` defines mobile `.hero-ctas` overrides
- `index.html` contains no element with class `hero-ctas` or `btn-primary` — confirmed by grep
- These were orphaned when hero CTAs were removed in commit `87c3d4e`

**Impact:** ~30 lines of dead CSS. No functional impact, but adds maintenance confusion.
**Fix:** Remove `.hero-ctas`, `.btn-primary`, and `.btn-primary:hover` rules from `styles.css`, including the mobile media query overrides.

### [LOW] F-07: No favicon

**Category:** Operations
**Evidence:**
- `index.html` has no `<link rel="icon">` tag — confirmed by grep for "favicon" and "icon"
- No `.ico`, `.png`, or `.svg` favicon files in the repo

**Impact:** Browsers show a generic tab icon. GitHub Pages will 404 on the default `/favicon.ico` request, generating noise in server logs.
**Fix:** Add a simple favicon (even a 1x1 transparent PNG or an SVG of "JB") and link it in the `<head>`.

### [LOW] F-08: No Content Security Policy

**Category:** Security
**Evidence:**
- `index.html` has no `<meta http-equiv="Content-Security-Policy">` tag
- No CSP headers configured (GitHub Pages doesn't support custom headers)

**Impact:** Low risk for a static site with no user input. However, if the site ever loads third-party scripts (analytics, widgets), there's no CSP to limit scope. The only external resource currently loaded is Google Fonts.
**Fix:** Add a CSP meta tag permitting `self`, Google Fonts origins, and `unsafe-inline` for styles:
```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src https://fonts.gstatic.com;">
```

### [LOW] F-09: CLAUDE.md nav list doesn't match actual nav

**Category:** Documentation
**Evidence:**
- `CLAUDE.md:27` says nav items are: "Home, Projects, About, GitHub (in-page anchors + external link)"
- Actual nav (`index.html:29-33`) has: Home, Projects, Process, About, GitHub — "Process" links to `#how-i-build`

**Impact:** Minor — CLAUDE.md is missing the "Process" nav item added during v2 enhancements.
**Fix:** Update CLAUDE.md nav list to include "Process" pointing to the How I Build section.

### [LOW] F-10: Typo in MD Reader card expanded content

**Category:** Consistency
**Evidence:**
- `index.html:153`: "I find I am using it seceral times a day" — "seceral" should be "several"

**Impact:** Visible typo in card detail content.
**Fix:** Change "seceral" to "several" on line 153 of `index.html`.
